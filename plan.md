# 💰 SAM (Account Manager)
> 회계 장부 데이터를 체계적으로 정리하고 분석하기 위한 프로젝트

---

## 🔐 Access & Security (역할 및 인증)

### 1. 사용자 역할 구분
| 구분 | 도메인 | 권한 | 핵심 기능 |
| :--- | :--- | :--- | :--- |
| **회계 관리자** | `sam.admin.javago.my` | Admin Code 보유 | 장부 수정, 액세스 코드 관리 |
| **회계 분석가** | `sam.javago.my` | 일반 Code 보유 | 장부 열람, 분석 도구 사용 |

### 2. 인증 메커니즘 (Access Code)
- **접속 방식**: 별도의 회원가입 없이 발급된 **Code**를 통해 접속
- **코드 관리**:
  - `Admin Code`: 유일하며, 유출 시 백엔드에서 직접 변경 (관리자 전용)
  - `일반 Code`: 관리자가 발급하여 분석가에게 공유
- **세션 관리**: JWT(JSON Web Token) 방식을 이용하며 일정 시간 동안 유지
- **접속 정책**: 편의성을 위해 단일 코드로 여러 기기에서 동시 접속 허용

## 📊 Data Architecture (데이터 구조)

### 1. 원본 데이터 속성 (Legacy)
현재 관리 중인 속성:
`날짜`, `분류`, `수입`, `지출`, `잔액`, `적요`, `영수 번호`, `결제인`, `송금`

### 2. 제안하는 데이터베이스 스키마 (Normalized)

#### 🧾 영수 테이블 (`receipts`)
영수증 기반의 핵심 거래 내역
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `id` | PK | 영수 고유 식별자 |
| `category_id` | FK | `분류` 테이블 연결 |
| `payer_id` | FK | `결제인` 테이블 연결 |
| `description` | String | 내역에 대한 상세 설명 (적요) |
| `income` | Decimal | 운영비 추가 등 수입 금액 |
| `expense` | Decimal | 소비 금액 (영수증 표기 금액) |
| `discount` | Decimal | 지원금으로 처리한 비용 |
| `people_count` | Integer | (선택) $n$명 발생 건의 정산용 인원 수 |
| `receipt_url` | String | Google Drive 원본 영수증 이미지 URL |
| `is_transferred` | Boolean | 결제인에게 송금 완료 여부 |
| `created_at` | DateTime | 장부 등록 시간 |
| `transaction_at` | DateTime | 실제 결제 처리 시간 |
| `transferred_at` | DateTime | 송금 완료 시간 |

> [!NOTE]
> **잔액 계산 공식**: `+수입 - 지출 + 할인`
> - 예: 10,000원을 결제하고 5,000원을 지원금으로 처리한 경우
> - `지출 = 10,000`, `할인 = 5,000`으로 기록하여 실제 잔액 차감은 5,000원이 됨.

#### 👥 결제인 테이블 (`payers`)
| 필드명 | 설명 |
| :--- | :--- |
| `id` (PK) | 결제인 식별자 |
| `name` | 결제자 이름 |
| `account` | 은행 계좌 정보 (기본값: Kakaopay) |

#### 📁 분류 테이블 (`categories`)
| 필드명 | 설명 |
| :--- | :--- |
| `id` (PK) | 분류 식별자 |
| `parent_id` (FK) | 부모의 분류 (Self-referencing) |
| `level` | 계층 깊이 (1~3단계 고정: 대/중/소) |
| `field` | 분류 이름 |

#### 📅 관리용 테이블
- **회계 기준 (`fiscal_terms`)**: 회계 연도, 시작일, 종료일 관리
- **예산안 (`budgets`)**: 분류별 책정 예산 및 회계 기준 연결
- **액세스 코드 (`codes`)**: 코드값 및 타입(admin/general) 저장

## 🚀 도메인 별 기능 구현

### 🛠️ 회계 관리자 (Admin)
- **인증**: 전용 도메인 접속 시 Admin Code 인증 후 관리 페이지 진입
- **데이터 마이그레이션**:
  - 기존 스프레드시트 데이터가 많지 않으므로 직접 복사/붙여넣기(수동 입력)를 통해 초기 데이터 구축
- **마스터 데이터 관리**:
  - 영수/결제인/Code 테이블 전반의 CRUD 기능
  - **분류 체계 관리**:
    - 추가/수정/삭제 가능 (삭제 시 연결된 영수는 `null` 처리하여 데이터 보존)
    - 분류 변경 시 연결된 영수에 연쇄 반영(Cascade)

### 📈 회계 분석가 (Analyst)
- **인증**: 일반 Code 인증 후 분석 대시보드 진입
- **주요 기능**:
  - **영수 내역 조회**: 기간별 필터링 및 상세 내역(영수증 이미지 포함) 확인
  - **소비 분석**:
    - **분류별 비중**: Pie Chart를 통한 지출 분포 확인
    - **지출 추이**: 월별 지출 변화 추적
    - **예산 대비 현황**: 예산안과 실제 지출액 비교 분석
